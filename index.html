Poisson ReLU Maps

What this website does. The website starts with an input point \(x\in\mathbb R^2\) and keeps every plot as a 2D view of that original input plane. It then applies four random-width ReLU maps \(G,H,I,J\). For each map \(M\in\{G,H,I,J\}\), the number of rows/features is random: \(N_M\sim\operatorname{Poisson}(\lambda)\). Thus \(G(x)\in\mathbb R^{N_G}\), \(H(x)\in\mathbb R^{N_H}\), \(I(x)\in\mathbb R^{N_I}\), and \(J(x)\in\mathbb R^{N_J}\). The graphs show the pullback of these random high-dimensional feature partitions back onto the original \(x_1,x_2\) plane.

\[ G(x)=\operatorname{ReLU}(sA_Gx-T_G),\qquad H(x)=\operatorname{ReLU}(sA_HG(x)-T_H),\qquad I(x)=\operatorname{ReLU}(sA_IH(x)-T_I),\qquad J(x)=\operatorname{ReLU}(sA_JI(x)-T_J). \]

For each \(M\in\{G,H,I,J\}\), the website samples rows \(A_{M,j}\) as unit vectors in the current feature dimension and offsets \(T_{M,j}\sim\operatorname{Uniform}([-a,a])\). The row scale \(s\) multiplies each sampled row inside the pre-activation. The plots stay in the original 2D input plane and show the pullback of these higher-dimensional ReLU partitions.

Controls

The seed is a reproducibility label. Once the seed and controls are fixed, the whole realization is fixed. A new seed starts a new pseudo-random stream, which is transformed into Poisson widths \(N_G,N_H,N_I,N_J\), unit rows \(A_{M,j}\), and offsets \(T_{M,j}\). This mimics the random process while still letting the same seed reproduce the same picture.

Each ReLU map samples \(N_G,N_H,N_I,N_J\sim\operatorname{Poisson}(\lambda)\). Larger \(\lambda\) usually means more rows/features.
This is the multiplier in each pre-activation, for example \(G_j(x)=\operatorname{ReLU}(sA_{G,j}x-T_{G,j})\). It changes how strongly each sampled row acts.
Offsets are sampled as \(T_{M,j}\sim\operatorname{Uniform}([-a,a])\) for \(M\in\{G,H,I,J\}\). The value \(a=0.5\) matches the unit-diameter ball convention.
Controls how the unit rows \(A_{M,j}\) are sampled before normalization.
Rejects very small Poisson widths so a layer is not almost empty.
Rejects very large Poisson widths so the plots stay readable.

Yellow boundary equations

This section lists the actual sampled row equations. Each row is a unit normal vector plus an offset. For the first map, the equations draw visible yellow lines directly in the original \(x_1,x_2\)-plane. For later maps, the rows cut the current feature space; the yellow boundary overlay is the pullback of that cut to the original 2D view.

For a map \(M\in\{G,H,I,J\}\), points with the same partition code lie in a cell of the same color. The partition code is defined by \[ c_M(x):=(c_1,\dots,c_{N_M}),\qquad c_j\in\{0,1\},\quad 1\le j\le N_M. \] Here \(c_j=1\) means the \(j\)-th pre-activation is positive, so that ReLU coordinate is active. \(c_j=0\) means the \(j\)-th pre-activation is negative or exactly on the boundary, so that coordinate is clipped to zero.
